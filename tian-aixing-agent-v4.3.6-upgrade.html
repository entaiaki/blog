天爱星Agent v4.3.5→v4.3.6

悬浮窗 3 层触摸模型重构：click/drag 彻底分家 + 双保险防误触关闭

2026-06-24

根因：GestureDetector + setOnTouchListener 混用

v4.3.4~v4.3.5 的触摸架构存在结构性问题：

  • setOnTouchListener 先拦截 → 检测 X 区域 → 剩余交 GestureDetector
  • GestureDetector.onSingleTapConfirmed() 有 300ms 延迟，体感卡顿
  • X 区域用坐标判断 → DOWN/UP 位置抖动 → 误关闭
  • 拖拽和点击在 GestureDetector 内混判，边界模糊

整条链路 3 个组件协作，任何一个抖动都会污染结果。

方案：标准桌宠 3 层模型

┌──────────────────────────┐
│ OverlayRoot              │
│                          │
│  ┌──── Close(X) ──────┐ │  ← isOnCloseArea() 精确区域 + 二次确认
│  └────────────────────┘ │
│                          │
│  ┌── Character Layer ─┐ │  ← 单击 → 打开悬浮聊天框
│  └────────────────────┘ │
│                          │
│  Background (drag only)  │  ← FloatingTouchController 惯性拖拽
└──────────────────────────┘

核心组件从 3 个（setOnTouchListener + GestureDetector + FloatingTouchController）精简为 1 个 OverlayTouchController：

class OverlayTouchController(
    onOpenChat      : () -> Unit,       // 非 X 区域单击
    onClose         : () -> Unit,       // X 区域二次确认后
    onCloseHint     : (() -> Unit)?,    // 第一次点 X → Toast
    onDragMove      : (Float,Float)->Unit,  // 拖拽位移
    onDragUp        : () -> Unit,       // 拖拽结束
    onLongPress     : (() -> Unit)?,    // 身体区域长按
    isOnCloseArea   : (Float,Float)->Boolean  // X 区域判定
)

防误触关闭（三保险）

保险机制阈值
#1 区域限制 isOnCloseArea(x,y) — 手指必须落在 X 按钮 30dp 区域内 局部坐标精确匹配
#2 明确点击 dx < 10px && dy < 10px && dt < 250ms — 短按+小移动才是点击 排除拖动和长按
#3 二次确认 第一次点 X → Toast "再点一次"；3 秒内在 X 区域再点 → 执行关闭 排除手滑和抖动 
fun onTouchEvent(event: MotionEvent): Boolean {
    when (event.action) {
        ACTION_DOWN -> { 记录位置+时间; floatTouch.onTouchEvent(event) }
        ACTION_MOVE -> { floatTouch.onTouchEvent(event); 检测长按 }
        ACTION_UP   -> {
            if (isClick) {
                if (isOnCloseArea) handleCloseClick()   // → 二次确认
                else onOpenChat()                       // → 悬浮聊天框
            } else {
                floatTouch.onTouchEvent(event)           // → 惯性动画
            }
        }
    }
}

设计原则

  1. 不用 clickable / setOnClickListener — 全部走 onTouchListener
  2. 点击 vs 拖拽在 UP 事件分叉 — 不是 onSingleTapConfirmed vs onScroll
  3. 关闭必须"明确且有意" — 两次点击 X 才能关，一次拖拽就取消待确认
  4. 桌宠 = Chat 入口 — 点击身体区域打开悬浮聊天框，行为和 ChatScreen 完全一致
