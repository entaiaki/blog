天爱星Agent 开发全记录：从零构建一个 Android AI 助手

2025年6月 · 预计阅读 15 分钟

记录了一个 Android Chat Agent 应用从零到最终 APK 的完整构建过程——9 次迭代打包、 38 个 Kotlin 源文件、4 张 Room 表、LLM 多提供商动态切换、AlarmManager 提醒系统、 以及 43 张可切换背景的 Material3 界面。

1. 起点：我们要构建什么

天爱星Agent 是一个运行在 Android 上的智能助手，能接入多种大语言模型 （DeepSeek / OpenAI / 通义千问 / 自部署），调用系统工具（打开应用、记事、天气、提醒）， 具备持久化记忆和多会话管理能力。架构上预留了 Live2D 虚拟角色和 TTS 语音合成接口。

层级 技术 选择理由
UIJetpack Compose + Material3声明式 UI，与 Kotlin 协程天然集成
架构AndroidViewModel + StateFlow生命周期安全，响应式数据流
数据库Room (SQLite)Google 官方 ORM，4 张表
HTTPOkHttp 4.12轻量，OpenAI 兼容 API
构建AGP 8.2.2 + Kotlin 1.9.22 + kaptGroovy Gradle
最低/目标API 26 / 34覆盖 95%+ 设备

2. 打包 #1 — 骨架搭建 (15.6 MB)

目标是创建一个能编译、能安装、能发送消息并收到 AI 回复的最小可行产品。一次性创建 25+ 文件，包括 PlannerAgent（JSON 工具调用）、LLMClient（OkHttp → DeepSeek）、4 个 Tool 实现、以及 Compose UI 层。

// PlannerAgent v1 核心调用逻辑
val toolJson = llmResponse.getJSONObject("tool_call")

val result = toolRegistry.execute(toolName, toolJson.getJSONObject("params"))
messages.add(JSONObject().apply {

踩坑修复：kotlinCompilerExtensionVersion 拼写错误、矢量图 <ellipse> 不兼容低版本 Android（改为 Path 贝塞尔曲线）、全屏深色主题配置。

3. 打包 #2 — 数据层 + 提醒通知

引入 Room 持久化，设计 4 张表：

user_facts(key, value)              -- 用户画像
conversations(id, title)            -- 会话列表
messages(conversation_id, role, content)  -- 消息历史
reminders(title, note, remind_at)        -- 提醒

提醒系统链路：ReminderTool.execute(JSONObject) → ReminderRepository.insert() → ReminderScheduler.schedule() → AlarmManager.setExactAndAllowWhileIdle() → ReminderReceiver → NotificationManager.notify()

Tool 接口经历了重要升级：属性从 fun name() 改为 val name + val description，入参从 Map<String, Any> 统一为 JSONObject。移除了 schema() 方法，降低实现复杂度。

4. 打包 #3 — ViewModel 响应式重构

ChatViewModel 从 ViewModel() 升级为 AndroidViewModel(application)，内部自管理 AgentDatabase 和 ConversationRepository，无需 MainActivity 手动注入依赖。

class ChatViewModel(application: Application) : AndroidViewModel(application) {
    val conversations: StateFlow<List<ConversationEntity>> =
        repo.allConversations.stateIn(viewModelScope, ...)

    private val _messages = MutableStateFlow<List<ChatMessage>>(emptyList())
    val messages: StateFlow<List<ChatMessage>> = _messages

    fun sendMessage(text: String) {
        viewModelScope.launch {
            _messages.update { it + ChatMessage("user", text) }
            _isLoading.value = true
            val reply = buildAgent().chat(text)
            _messages.update { it + ChatMessage("assistant", reply) }
            _isLoading.value = false
        }
    }
}

关键设计决策：ChatMessage(role, content)（UI 层 StateFlow）与 MessageEntity(conversationId, role, content)（Room 持久化）解耦，两层独立演进互不影响。

5. 打包 #4 — PlannerAgent 重写：文本态工具调用

v1 使用 JSON 格式要求 LLM 输出精确结构，但 DeepSeek 等模型常在 JSON 前后附加解释文字导致解析失败。重新设计为纯文本标记格式：

TOOL: add_reminder
PARAMS: {"title": "开会", "datetime": "2025-06-19 15:00"}

PlannerAgent 解析逻辑简化为逐行查找 TOOL: 和 PARAMS: 前缀：

repeat(3) {  // 最多 3 轮工具调用
    val toolName = response.lines()
        .find { it.startsWith("TOOL:") }
        ?.removePrefix("TOOL:")?.trim()
        ?: return response
    val params = JSONObject(response.lines()
        .find { it.startsWith("PARAMS:") }
        ?.removePrefix("PARAMS:")?.trim()
        ?: return response)
    val result = tools.find { it.name == toolName }
        ?.execute(params) ?: return response
    messages.add("tool" to "Tool result: $result")
    response = llmClient.chat(messages)
}

LLMClient 单例化：getInstance(context) 确保全局唯一 OkHttp 连接池被复用。

ChatScreen Material3 化：ModalNavigationDrawer + Scaffold + Surface 内联气泡 + OutlinedTextField 输入栏。清理 6 个废弃文件（AgentAction、PromptBuilder、ToolCall 等）。

6. 打包 #5 — 权限 + 启动图标

Android 13 (API 33) 起通知权限变为运行时权限，需要在 onCreate 中主动申请：

if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
    requestPermissionLauncher.launch(
        Manifest.permission.POST_NOTIFICATIONS)
}

启动图标使用 Python Pillow 从用户上传的 icon.png 生成 5 种密度 × 2 种形状 = 10 张 PNG（48×48 ~ 192×192），覆盖 mdpi 到 xxxhdpi。

7. 打包 #6 — Live2D/TTS 接口预留 + 背景系统 (~102 MB)

设计哲学：接口先行，实现后置。 Live2D SDK 和 TTS 引擎接入工作量大、依赖复杂，预留清晰接口避免未来重构。

interface Live2DController {
    val isReady: Boolean
    fun loadModel(modelPath: String)
    fun playMotion(group: String, index: Int, priority: Int)
    fun setExpression(expressionId: String)
    fun setLipSync(volume: Float)        // ← 与 TTS 联动
    fun setEyeFollow(x: Float, y: Float)
    fun release()
}

interface TTSController {
    val isSpeaking: Boolean
    fun speak(text: String,
        onPlaybackProgress: (Float) -> Unit = {},
        onStart: () -> Unit = {},
        onDone: () -> Unit = {})
    fun stop(); fun setRate(rate: Float)
    fun setPitch(pitch: Float); fun release()
}

联动链路：AI 回复 → TTSController.speak(onPlaybackProgress = { volume → live2DController.setLipSync(volume) }) → 音频播放时实时更新口型。

背景系统

BackgroundSource (sealed class)
├── Asset("bg_default_1.png")   ← 43 张预置图
├── Custom(uri)                 ← 相册选择 + persistableUriPermission
└── SolidColor                  ← 纯色兜底

BackgroundViewModel 用 SharedPreferences 持久化当前背景选择，提供 randomBackground() / setCustomBackground(uri) 方法。ChatBackground 用 Coil 加载 Custom 来源的本地 URI。底部弹出 BackgroundSettingsSheet 提供随机切换 / 相册选择 / 恢复默认三个入口。

8. 打包 #7 — ChatScreen v3 图层叠加

ChatScreen v2 的 Scaffold 使用不透明背景色，完全遮挡背景图。v3 通过三层叠加 + 半透明 alpha 值解决：

Box(fillMaxSize) {
    ChatBackground(source = background)    // 底层：背景图
    // Live2DComposable(...)                // 中层：角色（预留）
    Scaffold(containerColor = Color.Transparent) {  // 顶层
        TopAppBar(alpha = 0.85f)
        MessageBubble(alpha = 0.92f)
        ChatInputBar(alpha = 0.90f)
    }
}

5 个 ViewModel 依赖全部通过 Compose 默认参数注入——真实实现替换 NoOp 只需传参，零侵入。

9. 打包 #8 — 大模型动态切换

定义 LLMProvider 枚举统一 4 个提供商的 endpoint 格式：

enum class LLMProvider(val displayName: String,
    val baseUrl: String, val defaultModel: String) {
    DEEPSEEK("DeepSeek", "...deepseek.com/v1", "deepseek-chat"),
    OPENAI("OpenAI",     "...openai.com/v1",    "gpt-4o-mini"),
    QWEN("通义千问",     "...aliyuncs.com/...", "qwen-turbo"),
    CUSTOM("自定义",     "", "")
}

LLMConfigStore 将用户配置持久化到 SharedPreferences，LLMClient.applyConfig() 即时更新 endpoint/model/apiKey，无需重启应用。

LLMSettingsScreen 提供完整设置界面：提供商下拉选择、API Key（带显隐切换）、模型名称、Temperature 滑块、Max Tokens。从 ConversationDrawer 的新增「⚙️ 模型设置」入口进入。
