动机：App 内和桌宠应该是同一个人

v4.1 及之前，App 内的 Gal 布局和桌面悬浮窗各自维护自己的状态。 App 里的天爱星在微笑，桌面上的还在发呆——分裂感。

v4.2~v4.3 搭建了全局状态中心，让 App 内和桌宠共享同一个角色+情绪。 同时把桌宠从"能看不能摸"升级为完整的交互系统：点击弹聊天、长按出控制菜单、拖拽带惯性。

Core 1（v4.2）：全局状态中心

CharacterStateHolder — 单例 mutableStateOf

object CharacterStateHolder {
    var emotion: CharacterEmotion by mutableStateOf(CharacterEmotion.IDLE)
    var role: String        by mutableStateOf(CharacterPackRegistry.default.id)
    val currentPack: CharacterPack get() = CharacterPackRegistry.findById(role)
}

不同于 ViewModel 的 StateFlow，mutableStateOf 直接支持 Compose 重组 ——App 内 GalGameChatLayout 和桌宠 DesktopPetView 同时订阅，任一处修改立即双向同步。

EmotionBridge — LLM 情绪注入

object EmotionBridge {
    fun apply(emotion: CharacterEmotion) {
        CharacterStateHolder.emotion = emotion
    }
}

ChatViewModel 在 4 个情绪变化点（流式开始→THINKING、流结束→解析情绪、错误→IDLE、取消→IDLE）全都写入 CharacterStateHolder。

RoleManager — 委托 CharacterPackRegistry

object RoleManager {
    val roles: List<String> get() = CharacterPackRegistry.packs.map { it.id }
    fun switch(roleId: String) { CharacterStateHolder.role = roleId }
}

CharacterBackground — 封装 CharacterView

从 assets 动态加载，接收 role: String + emotion: CharacterEmotion。 不依赖 R.drawable，新增角色只需放图片 + 注册。

KokoroTTSBridge — 一调即播

object KokoroTTSBridge {
    fun init(context: Context) { ... }
    fun speak(text: String) { KokoroTTSManager.getInstance(ctx!!).feedStream(text) }
}

Core 2（v4.3）：桌宠交互系统

PetCommandCenter — 全局可见性

object PetCommandCenter {
    var isVisible by mutableStateOf(false)
    fun show()  { isVisible = true }
    fun hide()  { isVisible = false }
    fun toggle() { isVisible = !isVisible }
}

DesktopAgentService 的 ComposeView 内用 if (PetCommandCenter.isVisible) 包裹 DesktopPetView。 App 回到前台时 hide()，退后台时 show()，完全由状态驱动。

ChatOverlay — 悬浮聊天框

点击桌宠 → 全屏半透明遮罩 + 380dp 聊天卡片弹出。含顶栏（关闭按钮）、欢迎语区域、快捷按钮。

操控行为
点击桌宠ChatOverlayController.show() → 悬浮聊天框
点击关闭 / 外部ChatOverlayController.hide()
长按桌宠AlertDialog 三选一（关闭桌宠 / 取消 / 打开 App）

FloatingTouchController — 惯性拖拽

替代旧版 GestureDetector.onScroll 简单跟随。完整链路：

ACTION_DOWN  → 记录 rawX/rawY + 时间戳
ACTION_MOVE  → 累加 dx/dy → onMove 回调更新 WindowManager params
            → 计算速度 vx = dx/dt*16, vy = dy/dt*16
ACTION_UP    → 启动惯性线程：
               while (速度 > 1px && 帧数 < 60) {
                   vx *= 0.92;  vy *= 0.92
                   onMove(vx, vy)
                   Thread.sleep(16)
               }
               → snapToEdge() (预留)

关键设计：GestureDetector 仍负责点击/长按检测，FloatingTouchController 负责位移+惯性。 两者通过同一 setOnTouchListener 协作——GestureDetector.onTouchEvent 拿到事件后， FloatingTouchController 跟踪 MOVE 和 UP，互不干扰。
