项目背景

基于《AI智能招聘系统任务书 v1.0》，从零构建全栈招聘管理平台。核心需求：

  • 4 角色体系：管理员、HR、部门负责人、数据分析师
  • 9 节点状态机：投递 → AI打分 → 打分失败 → 已打分 → HR审核 → 部门审核 → 面试 → Offer → 入职/淘汰
  • 5 维度 AI 打分：技能匹配(25)、经验匹配(25)、教育匹配(20)、发展潜力(15)、稳定性(15)，总分 100
  • 操作审计日志：全链路可追溯

核心架构决策

1. LLM 配置数据库化

初始方案是 .env 硬编码，但遇到了 QwenPaw 桌面环境的系统环境变量冲突问题（OPENAI_BASE_URL 会覆盖 .env）。

解决：字段改用 LLM_ 前缀（LLM_BASE_URL/LLM_API_KEY/LLM_MODEL），并将配置存储到数据库 llm_configs 表。管理员通过前端页面在线管理，支持：

  • CRUD 多组配置
  • 测试连接（实时验证 API 可达性）
  • 一键激活/切换（AI 打分服务动态读取激活配置）

2. AI 打分服务三级容错

ai_service.py 实现了稳定的评分链路：

1. pdfplumber 解析 → 纯文本
2. 剥离 Qwen3 <think> 标签（正则 .*? 非贪婪匹配）
3. JSON 提取三级回退：
   L1: 直接 json.loads()
   L2: 正则提取第一个 {…} JSON 块
   L3: DEFAULT_SCORE 兜底（55/100，避免空值崩溃）

3. 权限依赖统一

重构前，5 个路由文件中散布着 14 处 if current_user.role not in [...]。重构后：

# deps.py — 一处定义，全局生效
def require_roles(*roles: UserRole):
    async def checker(current_user = Depends(get_current_active_user)):
        if current_user.role not in roles:
            raise HTTPException(403, ...)
        return current_user
    return checker

require_admin       = require_roles(UserRole.admin)
require_hr_or_above = require_roles(UserRole.admin, UserRole.hr)
require_any_staff   = require_roles(UserRole.admin, UserRole.hr, UserRole.dept_leader)

4. ApplicationService 服务层抽取

把投递业务的读写逻辑从路由层抽到独立的 Service 类：

class ApplicationService:
    def __init__(self, db: AsyncSession, operator_id: int)
    
    async def submit(job_id, candidate_id, resume_id) → Application
    async def transition(app_id, new_status, notes, interview_date) → Application
    async def retry_scoring(app_id) → Application
    async def save_scores(app_id, scores, summary, ...) → Application
    async def mark_scoring_failed(app_id, error) → Application

路由层只做 HTTP 胶水：参数校验 → 调 Service → 异常映射为 HTTP 状态码。

踩坑记录

  • OPENAI_BASE_URL 环境变量冲突：QwenPaw 桌面自动设置了该变量，覆盖 .env，导致 AI 打分连接到错误端点。改用 LLM_ 前缀解耦。
  • pdfplumber vs PyPDF2：PyPDF2 对中文 PDF 解析效果差，切换到 pdfplumber 后准确率大幅提升。
  • Qwen3 <think> 标签：部分 Qwen 模型返回带 <think> 标签的推理过程，JSON 解析前需要先剥离，否则 json.loads 失败。
  • FastAPI Depends 参数顺序：当路由同时有 Form() 和 Depends() 时，Depends 必须放在 Form 之后，否则参数绑定会失败。
  • aiosqlite 测试隔离：每个测试用 scope="function" + create_all/drop_all 确保完全隔离，避免了数据污染问题。
