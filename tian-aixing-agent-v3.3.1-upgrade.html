天爱星Agent v3.3.1 桌宠动画重构

悬浮窗动画系统 · 立绘组件重写 · 情绪角标 · 三层独立动画管线

2026.06.24

动机：桌宠「不像活的」

v3.3 的桌面悬浮窗虽然是前台 Service + Compose，但立绘几乎不动——仅一个上下浮动， 且图片加载依赖 getIdentifier 运行时反射，缺失文件时默默降级占位符， 用户甚至察觉不到图片没加载成功。

这次 v3.3.1 的目标很明确：让桌面上的天爱星「活起来」。

变更文件清单

文件动作说明
OverlayCharacter.kt重写呼吸浮动 + 左右摇摆 + 身体微晃 + 16 情绪角标 + 底部标签
CharacterView.kt重写硬编码 R.drawable + talkBob 口型动画 + slideIn 情绪切换
DesktopAgentService.kt修改overlay 尺寸 200×340 → 90×160，移除冲突 Compose clickable

1. OverlayCharacter 悬浮窗 — 三层动画管线

旧版 OverlayCharacter 仅一个 floatY 动画，现在拆成三条独立叠加的循环： 

floatY   : 0 ↔ -6dp,  2400ms, FastOutSlowInEasing  (呼吸)
swayX    : 0 ↔  3dp,  3200ms, CubicBezier(0.4,0,0.6,1)  (左右摇摆)
swayRot  : -0.8° ↔ 0.8°, 3200ms, CubicBezier          (身体微晃)

三条动画周期不同（2400 vs 3200），相位自动错开，看起来像自然体态而非机械重复。 叠加在同一个 graphicsLayer { translationY, translationX, rotationZ } 上， 无需额外开销。

2. CharacterView 立绘系统 — 从反射到硬编码

旧问题

  • getIdentifier 运行时按名称查找 R.drawable，缺失时返回 0 → 显示占位方块
  • 情绪切换仅 fadeIn/fadeOut，没有空间位移，视觉上像「闪切」
  • 说话时 floatY 既做呼吸又做口型，两种频率打架
  • 仅 7 种情绪，缺少悲伤/哭泣/沮丧/困乏等日常表情

新设计

// 16 种情绪全部硬编码映射，缺图直接编译报错 → 开发期就能发现

    CharacterEmotion.SLEEPING   -> R.drawable.tianai_sleeping
    CharacterEmotion.SOBBING    -> R.drawable.tianai_sobbing
    CharacterEmotion.CRYING     -> R.drawable.tianai_crying
    CharacterEmotion.DEPRESSED  -> R.drawable.tianai_depressed
    CharacterEmotion.DISTRESSED -> R.drawable.tianai_distressed
    CharacterEmotion.DROWSY     -> R.drawable.tianai_drowsy
    CharacterEmotion.SWEATING   -> R.drawable.tianai_sweating

}

三层独立动画

floatY     : 呼吸浮动  0 ↔ -6dp (talking: 400ms / idle: 2200ms)
talkBob    : 口型微动  0 ↔ -4dp (talking only: 280ms)
talkScale  : 说话脉冲  1 ↔ 1.018 (talking: 280ms / idle: 600ms)

// 叠加：translationY = floatY + talkBob
// 口型和呼吸互不干扰，各自独立周期

情绪切换动画升级

transitionSpec = {
    (fadeIn(tween(320)) + slideInVertically(
        animationSpec  = tween(320, easing = FastOutSlowInEasing),
        initialOffsetY = { it / 14 }    // 新图从下方 7% 处滑入
    )).togetherWith(
        fadeOut(tween(240))              // 旧图 240ms 淡出
    )
}

新图不是凭空出现，而是从略偏下方滑入 + 淡入，配合旧图同时淡出。 视觉上像角色「调整姿势」而非「切换图片」。

3. DesktopAgentService — 尺寸与交互修正

旧 overlay 尺寸 200×340dp，但 OverlayCharacter 实际内容仅 90×160dp。 Window 比内容大 2 倍以上 → 触摸区域有大量空白死区 → 拖拽和点击体验差。 

// 修正
- dpToPx(200), dpToPx(340)   →   dpToPx(90), dpToPx(160)

同时移除了 OverlayCharacter 的 .clickable {}，因为 Compose 的 clickable 会拦截 MotionEvent，导致 setOnTouchListener 的拖拽逻辑失效。 现在交互职责清晰分离： 

WindowManager.setOnTouchListener  →  拖拽移动 + 点击回 MainActivity
Compose OverlayCharacter          →  纯视觉：浮动+摇摆+立绘+角标
