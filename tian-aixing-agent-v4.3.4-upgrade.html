天爱星Agent v4.3.3→v4.3.4

两层事件分流：X 与身体物理隔离，彻底消灭误关闭

2026-06-24

根因：手势系统混用导致事件污染

v4.3.3 的问题是 GestureDetector 统一拦截所有触摸事件，然后用 isCloseButtonHit(e) 坐标判断是 X 还是身体。坐标抖动 → 误判 → stopSelf() 杀掉服务。

v4.3.4 改为两层分流、物理隔离：

composeView.setOnTouchListener { _, event ->
    when (event.action) {
        ACTION_DOWN -> {
            touchStartedInCloseZone = isInCloseZone(event)
            if (touchStartedInCloseZone) return@setOnTouchListener true  // ← X 区域，消费
        }
        ACTION_UP -> {
            if (touchStartedInCloseZone) {
                if (isInCloseZone(event)) dismissPetWindowOnly()  // ← 只移除窗口
                touchStartedInCloseZone = false
                return@setOnTouchListener true
            }
        }
    }
    // ↓ 非 X 区域，走原有手势系统
    gestureDetector.onTouchEvent(event)
}
对比v4.3.3v4.3.4
手势系统GestureDetector 包揽一切View 层先分流，GestureDetector 只处理身体
X 判断onSingleTapConfirmed 内坐标猜ACTION_DOWN 阶段就分流
关闭行为hidePet + stopSelfdismissPetWindowOnly（不杀服务）
误关闭风险高（坐标漂移）无（物理隔离）
