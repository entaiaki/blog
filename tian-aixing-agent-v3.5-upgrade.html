动机：告别等待，边收边说

v3.4 及之前，LLM 回复要等完整返回后才显示并 TTS 播放。用户体验是一段长时间空白 → 突然一大段文字 + 语音。 v3.5 的核心目标：LLM 边吐字边显示（打字机效果） + 边合成 TTS 边播放。 同时加入折叠屏适配、立绘系统精简、TTS 断句逻辑重写。

1. LLM 流式接口

// 旧：同步等待完整响应
val reply = llm.chat(messages)  // 阻塞 5~30s

// 新：Flow 流式
llm.chatStream(messages).collect { chunk ->
    fullText += chunk
    updateMessageUI(fullText)     // 实时打字机效果
    ttsManager.feedStream(chunk)  // 边收边播
}

底层使用 OkHttp 长连接读 SSE（Server-Sent Events），逐行解析 data: {"choices":[{"delta":{"content":"..."}}]}， 每收到一个 token 就 emit 到 Flow。解析失败不中断流。

2. ChatViewModel 流式管道

send(userInput)
  ├─ 保存用户消息到 DB + UI
  ├─ 添加空 assistant 占位消息
  ├─ llm.chatStream(history).collect { chunk →
  │    ├─ fullText += chunk
  │    ├─ 实时更新 _messages[last]（过滤 [EMOTION:xxx]）
  │    └─ ttsManager.feedStream(cleanChunk)  // 凑够一句自动播
  │  }
  ├─ EmotionParser.parse(fullText) → 情绪立绘 + 悬浮窗
  ├─ ttsManager.flushStream() → 播放缓冲区剩余
  └─ DB 保存完整回复

stopGeneration() 支持中途取消：streamJob?.cancel() + ttsManager.stop()。 加载中按钮旋转 90° 变 × 图标。

3. 折叠屏适配

检测逻辑

// MainActivity.detectFolded()

// 内屏 2352×2172 → ratio ≈ 1.08 → isFolded = false
// 外屏 2376×1060 → ratio ≈ 2.24 → isFolded = true
return ratio > 1.6f

configChanges 声明 10 项配置变化（orientation|screenSize|...）， 折叠/展开不重建 Activity，onConfigurationChanged 即时更新 isFolded → Compose 自动重组。

4. CharacterView 精简

旧（v3.4）：内部自带动画
  - CharacterView 内含 infiniteTransition → floatY + talkScale + talkBob
  - AnimatedContent 情绪切换
  - 约 100 行

新（v3.5）：纯渲染组件
  - 仅 Image + contentScale + BottomCenter 对齐

资源命名同步改为 tianaixing_*.png，16 个文件全部重命名。 contentScale 参数由外层传入，内屏 Fit、外屏 Crop。

5. KokoroTTSManager 三级断句

feedStream(chunk) {
    streamBuffer.append(chunk)

    ① 主断句：遇到 。！？…\n.!? → 立即切一句
    ② 次级断句：遇到 ，；：,;: 且积累 ≥ 15 字 → 在逗号处切
    ③ 强制切断：buffer > 50 字仍无标点 → 截断前 50 字
}

缓冲区改用 StringBuilder，播放队列用 consumeQueue() 协程循环替代递归 processNext()， suspendCancellableCoroutine 支持取消传播。 TTS 合成由 KokoroTTS.synthesizeToFile → MediaPlayer 播放。
