天爱星Agent v3.6：背景套系系统 & 悬浮窗修复 & 生命周期治理

6 套主题背景切换 · 套系内精细化选图 · 悬浮窗可见性 & 生命周期解耦 · Gal 布局背景图真正接入 · onPause/onStop 竞态修正

2026.06.24

动机：背景不应只是一个纯色

v3.5 及之前，Galgame 布局的"背景"实际上被 @Suppress("UNUSED_PARAMETER") 忽略了—— 一直是个纯黑渐变兜底。用户早已可以在设置里切换背景，但由于 GalGameChatLayout 从未消费 background 参数，无论怎么选都看不到效果。

v3.6 一次性解决三个问题：背景图真正渲染、套系系统（六套主题各有 6 张图）、 以及 悬浮窗与 Activity 生命周期的精准治理。

Core 1：背景套系系统

BackgroundViewModel — 套系定义与选择

新增 data class BackgroundTheme，包含 id / name / emoji / folder / fileNames。 六大套系各自拥有 6 张图片（存放在 assets/backgrounds/{folder}/ 下）：

套系emoji文件夹图片
深夜星空🌌night/night_1~6.png
樱花物语🌸sakura/sakura_1~6.png
深海之境🌊ocean/ocean_1~6.png
幽静森林🌿forest/forest_1~6.png
赛博霓虹🌃cyberpunk/cyberpunk_1~6.png
纯色渐变🎨—无图片，渐变色兜底

新增 API：

  • selectTheme(theme) — 切换到指定套系，随机选图
  • randomInCurrentTheme() — 在当前套系内随机换图（避开当前）
  • selectSpecificAsset(path) — 选中套系内的指定图片
  • currentTheme: StateFlow<BackgroundTheme> — 当前套系响应式状态
  • 旧 randomBackground() 保留为 randomInCurrentTheme() 的别名，完全向后兼容

BackgroundSettingsSheet — 套系选择 UI

从旧的回调式弹窗（onRandomBackground / onCustomBackground / onResetDefault） 重构为直接依赖 BackgroundViewModel 的 ModalBottomSheet：

  • LazyRow 卡片：六套系 72dp 渐变圆角方块 + emoji + 选中白边框 + 右上角 ✓
  • LazyVerticalGrid 缩略网格：当前套系 6 张图片的 3 列预览，点击直接切换
  • "随机换一张" 按钮：在当前套系内随机
  • Coil AsyncImage：从 assets/backgrounds/ 加载缩略图
  • 相册自选 + 恢复默认

Core 2：GalGameChatLayout 背景真正渲染

移除 @Suppress("UNUSED_PARAMETER")，新增 GalBackground 私有 composable， 根据 BackgroundSource 枚举分发渲染：

BackgroundSource渲染方式
Asset(fileName)Coil 加载 file:///android_asset/backgrounds/{fileName}，ContentScale.Crop
Custom(uri)Coil 加载用户自选 URI
SolidColor三层垂直渐变纯色兜底

布局层次（从底到顶）重构为 5 层：
背景图 → 半透明遮罩（0.25α 黑色覆盖，提升文字可读性）→ 立绘（GalBackgroundCharacter）→ 渐变蒙层（底部渐黑）→ UI 层

Core 3：悬浮窗 & 生命周期精准治理

DesktopAgentService — 可见性与启动解耦

Action旧行为新行为
ACTION_START操作 View 可见性 + startForeground（逻辑纠缠）仅确保 foreground 存活，View 首次创建默认 GONE
ACTION_SHOW—新增：View.VISIBLE
ACTION_HIDE—新增：View.GONE

hideOverlay() 改用 startService 而非 startForegroundService， Service 未运行时不崩溃。

MainActivity — onPause/onStop 竞态修正

生命周期旧行为新行为
onResumehideOverlay + 权限检查不变
onPauseshowOverlay（可能与 onStop 的 start 冲突）空操作
onStop仅 start（"双保险"）start + showOverlay，统一管理后台悬浮窗唤醒

旧逻辑：onPause 调 showOverlay → 紧接着 onStop 调 start 可能重置 View 状态。新逻辑：所有悬浮窗操作集中于 onStop，先确保 Service 活，再显示 View。

技术细节

Coil 加载 assets 图片

// BackgroundSettingsSheet 缩略图
ImageRequest.Builder(context)
    .data("file:///android_asset/backgrounds/$assetPath")
    .crossfade(true)
    .build()

// GalBackground 全屏背景
AsyncImage(
    model = ...,
    contentScale = ContentScale.Crop,
    modifier = Modifier.fillMaxSize()
)

套系持久化

// 保存套系 ID + 当前图片路径
prefs.edit()
    .putString(KEY_THEME_ID, theme.id)  // 套系
    .putString(KEY_ASSET, path)         // 图片
    .apply()

重启后自动恢复到上次套系 + 上次图片。

Activity 生命周期非对称处理

onResume  → hideOverlay     // 回到前台：收悬浮窗
onPause   → (no-op)          // 不抢 onStop 的活
onStop    → start + show     // 退到后台：先保活，再显示
