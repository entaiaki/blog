一、LLM 请求地址自动补全

之前 LLMConfig.endpoint 直接返回原始 URL。用户填 https://api.openai.com/v1，就 POST 到它——缺少 /chat/completions 后缀，API 返回 404。

val endpoint: String
    get() {
        val raw = (if (provider == LLMProvider.CUSTOM) customUrl
                   else provider.baseUrl).trim().trimEnd('/')

        return if (raw.endsWith("/chat/completions")) raw
               else "$raw/chat/completions"
    }

现在无论填 /v1 还是 /v1/chat/completions，都保证发到正确地址。

二、chatStream() — 移除 runBlocking

旧新
循环reader.forEachLine { }while(true) { reader.readLine() }
emitrunBlocking { emit(content) }emit(content) 直接调用
delta 取值.getJSONObject("delta")（NPE 风险）.optJSONObject("delta")（安全）

runBlocking 在 Flow 协程内硬堵线程，会导致流式输出卡死或 UI 永远收不到回复。直接 emit 走 Flow 内建通道，稳且快。

三、桌宠 X 按钮坐标精准判断

v4.3.2 用 IconButton——Material 默认最小触控区远大于 26dp，点身体也触发关闭。

v4.3.3 改为：

  • Compose 层：移除 pointerInput + IconButton，X 用纯 Box 只画图标
  • Service 层：GestureDetector 用 isCloseButtonHit(e) 按坐标判断
fun isCloseButtonHit(e: MotionEvent): Boolean {
    val closeSize = dpToPx(30)
    return e.x >= composeView.width - closeSize && e.y <= closeSize
}

override fun onSingleTapConfirmed(e: MotionEvent): Boolean {
    if (isCloseButtonHit(e)) {
        hidePet(); stopSelf()
    } else {
        InteractionBridge.requestChat()
    }
    return true
}

只有手指确实落在右上角 30px 范围内才关闭桌宠，其余全部弹出聊天框。

四、悬浮窗聊天历史去重

ChatOverlay 发送消息时先 messages.add(userMessage)，再构建 history 发给 PlannerAgent。但 PlannerAgent.chat() 内部也会把当前消息加一次 → 用户消息在 context 里出现两次。

// ❌ 旧
val history = messages.map { ... }

// ✅ 修
val history = messages
    .dropLast(1)
    .filter { it.content.isNotBlank() }
    .map { ... }
