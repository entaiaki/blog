动机：立绘为什么不该压在背景图上？

v3.5~v3.6，Gal 布局是 5 层结构：套系背景图 → 半透明遮罩 → 立绘 → 渐变蒙层 → UI。

问题是：背景图抢戏。樱花纷飞、霓虹闪烁确实好看，但它们把立绘压成了"配菜"。 一张 png 角色站在一张风景照上，不像 Galgame，像 PS 新手练手。

真正的视觉小说只会用纯色 / 模糊 / 极简背景。主角永远是角色自己。 v4.1 做了这个减法——删掉背景图+遮罩两层，让立绘在深夜蓝紫渐变上成为唯一的视觉焦点。 与此同时，v4.0 把角色系统从硬编码单角色扩展到了注册表式多角色包。

Core 1（v4.0）：多角色包系统

旧的 CharacterView 硬编码 R.drawable.tianaixing_idle，新增角色需要改 Kotlin 代码 + 放 drawable 资源。v4.0 彻底重写：

CharacterPack — 注册表式角色定义

data class CharacterPack(
    val id       : String,   // 唯一标识，如 "tianaixing"
    val name     : String,   // 天爱星
    val folder   : String,   // assets/characters/{folder}/
    val emoji    : String,   // 🌟
    val labelJa  : String    // 天愛星（あい）
)

object CharacterPackRegistry {
    val packs = listOf(
        CharacterPack("tianaixing", "天爱星", "tianaixing", "🌟", "天愛星"),
        CharacterPack("hoshino_ai", "星野爱", "hoshino_ai", "💜", "星野アイ"),
        // 新增角色只需加一行数据 + 放图片到 assets
    )
    fun findById(id: String) = packs.find { it.id == id }
}

CharacterView — 动态路径加载

// 旧：硬编码 R.drawable
Image(painter = painterResource(R.drawable.tianaixing_idle), ...)

// 新：从 assets 动态加载
val path = "file:///android_asset/characters/${pack.folder}/${emotion.name.lowercase()}.png"
AsyncImage(model = path, contentDescription = emotion.name, ...)

图片路径规范：assets/characters/{folder}/{emotion_name_lowercase}.png。 不支持的情绪自动回退 idle.png，图片缺失回退 Canvas emoji 占位。

CharacterPackSheet — 角色切换 UI

ModalBottomSheet 卡片式选择器：120×180 角色预览图 + 渐变底部遮罩 + 日文标注 + 选中紫色边框 ✓。
角色切换触发 AnimatedContent 过渡——key 改为 Pair(pack.id, emotion)， 换角色和换情绪共用同一套动画。

持久化

CharacterPackManager 使用独立 SharedPreferences("character_pack") 持久化当前角色， 与 LLMConfigStore 模式一致。

GalTypingIndicator — 动画精简

// 旧：三个独立 animateFloat + delayMillis 模拟波浪
val dots = listOf(
    animateFloat(..., delayMillis=0),
    animateFloat(..., delayMillis=200),
    animateFloat(..., delayMillis=400)
)

// 新：一个 animateFloat + StartOffset(200ms 间隔)
repeat(3) { index ->
    animateFloat(..., initialStartOffset = StartOffset(index * 200))
}

从 19 行压到 12 行，语义更清晰，点大小从 8dp → 6dp 更精致。

ChatInputBar → GalInputBar — 去动画化

旧 ChatInputBar 包含发送按钮的旋转/缩放/颜色动画、loading 状态切换的关闭按钮—— 在只有 SSE 流式回复的场景里纯属过度设计。新 GalInputBar：

  • 参数从 4 个减为 3 个（去掉 isLoading）
  • 发送按钮改为条件激活：无输入时灰色 0.15α，有输入时 AccentPurple
  • 所有 LaunchedEffect / animateFloatAsState / spring 移除
  • 新增 navigationBarsPadding() 适配全面屏
  • 输入框容器色改为 0x44000000 / 0x33000000 半透明而非纯透明
