天爱星Agent v3 升级日志：动画革命 & 设置系统重写

2026年6月21日 阅读约 8 分钟

概述

在前文 天爱星Agent 开发全记录 中， 我们构建了一个多 LLM 提供商、Tool 工具调用、Room 持久记忆的 Android AI 助手。 v3 系列升级聚焦于全量动效重写和设置系统模块化， 将 38 源文件扩充到 42+，每轮迭代坚持零警告构建。

动画令牌（AnimTokens）语义化

v2 中动画参数是散落的裸 Float 和 Int： 55L、0.5f、400f…… 全局搜索替换时极易误改。v3 将它们收敛到 object AnimTokens， 用 Compose Spring.DampingRatio* / Spring.Stiffness* 常量 替代裸 float，并按场景细分： 

object AnimTokens {
    const val StaggerBase     = 55L    // 抽屉列表每项延迟 (ms)
    const val MessageStagger  = 25L    // 消息列表每项延迟 (ms)
    const val MessageSlideInY = 28f    // 新消息弹入起始偏移 px
    const val FadeDuration    = 220    // 通用淡变时长 (ms)

    const val BouncyDamping   = Spring.DampingRatioMediumBouncy  // 气泡/按钮
    const val BouncyStiffness = Spring.StiffnessMedium
    const val SnapDamping     = Spring.DampingRatioLowBouncy     // 快速响应
    const val SnapStiffness   = Spring.StiffnessHigh
    const val DrawerDamping   = Spring.DampingRatioMediumBouncy  // 抽屉
    const val DrawerStiffness = Spring.StiffnessMediumLow
}

这样新增动画时只需引用语义名，全局调参也只需改一处。

SplashScreen Canvas 粒子启动屏

完全用 Compose Canvas 重写：70 个粒子飘移 + 双层呼吸光圈—— 大圆 300dp / blur 48dp，小圆 140dp / blur 28dp 反相脉动。 标题逐字 65ms，副标题逐字 55ms，闪烁光标 ▋。 零外部依赖，纯 Compose 原生绘制。

ChatScreen v3.1 全量动效重写

SharedTransition 页面切换

将原来 showReminder / showSettings 两个 boolean 升级为 Screen 路由枚举（Chat / Reminder / Settings）： 

SharedTransitionLayout {
    AnimatedContent(
        targetState = currentScreen,
        transitionSpec = { slideInHorizontally + fadeIn with bouncy spring }
    ) { screen ->
        sharedElement("conv_title_$id") { ... }
    }
}

Chat → 子页面右滑入 + 淡入，标题栏共享元素过渡。

AnimatedMessageBubble 打字机输出

新建独立 AnimatedMessageBubble.kt，从下方弹入 + 逐字输出：

字符类型延迟
中文句号/问号/感叹号55ms
中文逗号20ms
换行80ms
普通字符10ms

LLM 思考中显示 ThinkingDots——三点逐波跳动（rememberInfiniteTransition，150ms stagger）。

ChatBackground 背景系统升级

Asset/Custom 来源统一叠加暗色蒙层（0x99000000）。 默认模式改用纵向四段渐变 + 右上角 220dp 紫色光晕（blur 80dp） + 左下角 180dp 蓝色光晕（blur 70dp），层次感大幅提升。

ConversationDrawer 玻璃态重写

  • 纵向渐变背景 + 右上角紫色径向光晕
  • NewConversationButton 紫色边框 + 按下缩小 0.96x snap
  • StaggeredDrawerItem 从左侧滑入（-30px）+ 淡入
  • ConversationItem 左侧紫色选中指示条 animateDpAsState
  • 图标从 emoji 升级为 Material Icons（Notifications / Settings）

LLMSettingsScreen 设置系统重写

这是 v3 升级中最大的架构变更：之前 LLM 配置直接存储在 SharedPreferences 中，UI 由旧组件拼凑。 现在完全模块化：

LLMSettingsViewModel

新建 LLMSettingsViewModel（AndroidViewModel）， 持 LLMSettings data class 7 字段的 StateFlow：

data class LLMSettings(
    val apiKey      : String  = "",
    val baseUrl     : String  = "",
    val modelName   : String  = "",
    val temperature : Float   = 0.7f,
    val maxTokens   : Int     = 2048,
    val stream      : Boolean = true,
    val contextEnabled : Boolean = true
)

save() 方法自动推断 provider（deepseek / dashscope / custom）， 写入 LLMConfigStore + 即时 LLMClient.updateConfig()。

LLMSettingsScreen UI

全量替换旧设置页，按统一动画语言重写：

  • SettingsTextField — 边框发光动画（有内容 0.75α / 空 0.25α）
  • SettingsSlider — 玻璃卡片 + 实时数值显示，Int / Float 自适应格式
  • SettingsSwitch — 整行弹性缩放反馈 + Switch 直挂 onCheckedChange
  • SaveButton — 按下缩放 + animateColorAsState 颜色脉冲（紫→绿 "✅ 已保存"），1.5s 自动恢复
  • AnimatedBackButton — 独立组件，按下 0.88x snap 弹簧

所有设置项通过 StaggeredSettingsItem 统一 stagger 弹入（index × 55ms + 80ms）。

BackgroundSettingsSheet 动效补全

已有的背景设置底部弹出在本次升级中补全了动效层：

  • GradientPresetCard — 6 种渐变预设色卡，scale 0.7→1 弹入 + 点击 0.9x 缩放 + 外发光边框动画
  • ActionRow — 从 Surface 改为纯 Row + 背景高亮反馈（bgAlpha 0→0.15），缩放 0.97x
  • StaggeredSheetItem — 操作入口 stagger 延迟 index × 55ms + 100ms

ReminderScreen 动效重写

提醒页从简单列表升级为全动效页面：

  • 渐变背景 + 页面淡入 + FAB 弹簧缩放入场
  • AnimatedReminderCard 带 Checkbox + 完成态右移+淡出 + 删除按钮按下缩放 0.8x
  • AddReminderDialog 弹窗弹入动画（scale 0.85→1 + alpha 0→1）
  • 自由文本时间输入 + ReminderViewModel 内置智能解析（标准格式 / MM-dd HH:mm / 默认+1h）

Compose BOM 升级

compose-bom:2024.02.00 → 2024.09.03， 解锁 Compose 1.7.x 弹簧动画 API（Spring.DampingRatio*、Spring.Stiffness*）， 仅 1 个 deprecation（menuAnchor() 待后续迁移）。
