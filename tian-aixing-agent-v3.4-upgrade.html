天爱星Agent v3.4：立绘背景一体化 & 前台/后台联动

GalGameChatLayout 架构简化 · DesktopAgentService 可见性切换 · MainActivity 生命周期接管 · GalMessageBubble 打字机动画

2026.06.24

动机：三层变两层，前台无悬浮窗

v3.3/v3.3.1 的聊天界面有 ChatBackground（独立背景组件） + CharacterView（立绘组件） 两层叠加， 架构冗余且立绘存在感不够。同时悬浮窗在 App 前台和后台都显示，用户聊天时右下角还有一个桌宠——多余。

v3.4 做两件事：立绘直接当全屏背景、前台隐藏悬浮窗/后台显示。

变更文件清单

文件动作说明
GalGameChatLayout.kt重写立绘直接当全屏背景，移除 ChatBackground 层
DesktopAgentService.kt重写View.VISIBLE/GONE 切换替代 add/remove，新增 SHOW/HIDE action
MainActivity.kt重写onResume→hideOverlay / onPause→showOverlay 统一接管
ChatScreen.kt清理移除悬浮窗 LaunchedEffect + AlertDialog，由 MainActivity 集中管理

1. 立绘背景一体化

旧 4 层：底色 → ChatBackground → 渐变蒙层 → UI
新 4 层：底色 → GalBackgroundCharacter（立绘满屏）→ 渐变蒙层 → UI
                                    ↑
                  立绘就是背景，不再需要 ChatBackground 组件

GalBackgroundCharacter 内联 CharacterView，充满全屏、底部对齐、padding(bottom=64dp) 防止脚被输入栏遮挡。 情绪切换时 AnimatedContent 交叉淡入淡出 + slideIn 从下方 10% 进入。

2. 前台/后台联动

App 启动     → startForegroundService() → 创建 View 但设 GONE
App 在前台   → View.GONE                  用户看聊天界面
按 Home 键   → onPause()                   → View.VISIBLE  桌宠出现
切回 App    → onResume()                  → View.GONE     桌宠消失

关键：使用 View.VISIBLE/View.GONE 切换，不销毁 Compose 树，零闪烁。

MainActivity 生命周期接管

onResume()  → DesktopAgentService.hideOverlay()  + 权限检查
onPause()   → DesktopAgentService.showOverlay()
onStop()    → DesktopAgentService.start()          （双保险）

DesktopAgentService 新增 Action

ACTION_SHOW  → overlayView.visibility = View.VISIBLE
ACTION_HIDE  → overlayView.visibility = View.GONE
ACTION_START → showOverlay() + visibility = GONE（默认隐藏）

3. GalMessageBubble 打字机动画

弹入动画：spring(BouncyDamping, BouncyStiffness) + translateY 24→0
打字机逐字：中文标点智能延迟
    。！？.!? → 50ms
    ，,；;   → 18ms
    \n       → 70ms
    其他     → 9ms
气泡宽度：fillMaxWidth(0.62f) + widthIn(max=230dp)

4. ChatInputBar 增强

加载中：按钮旋转 90° 变 × 图标（Icons.Default.Close）
有文字：btnScale=1.0, btnTint=AccentPurple, borderAlpha=0.75
空文字：btnScale=0.92, btnAlpha=0.35, borderAlpha=0.2
maxLines=4 支持多行输入
navigationBarsPadding() 适配全面屏手势
