天爱星Agent v4.3.4→v4.3.5

折叠屏检测重写：ratio hack → Jetpack WindowInfoTracker

2026-06-24

背景：ratio hack 的局限性

v4.1~v4.3.4 的折叠屏检测使用 width/height ratio > 1.6f 猜测：

// v4.3.4 旧实现
val isFoldedState = remember {
    val ratio = context.resources.displayMetrics.widthPixels.toFloat() /
                context.resources.displayMetrics.heightPixels.toFloat()
    mutableStateOf(ratio > 1.6f)
}

三个问题：

  1. 平板横屏误判 — 8 寸平板横放 ratio 也可能 > 1.6，误触折叠模式
  2. 折叠半开漏判 — HALF_OPENED 状态时 ratio 可接近 1.0，UI 没切
  3. 不响应运行时折叠 — 用户边用边折叠时，ratio 不变 → 布局僵住

方案：Jetpack WindowInfoTracker

Google 官方 androidx.window 库的 WindowInfoTracker + FoldingFeature 直接读取 HAL 层折叠状态，零猜测。

// build.gradle 新增依赖
implementation "androidx.window:window:1.2.0"
// MainActivity.kt — 折叠状态观察
private fun observeFoldingState() {
    lifecycleScope.launch {
        WindowInfoTracker.getOrCreate(this@MainActivity)
            .windowLayoutInfo(this@MainActivity)
            .collectLatest { layoutInfo ->
                val foldingFeature = layoutInfo.displayFeatures
                    .filterIsInstance<FoldingFeature>()
                    .firstOrNull()

                isFolded.value = if (foldingFeature != null) {
                    foldingFeature.state != FoldingFeature.State.FLAT
                } else {
                    fallbackDetectFolded()  // 非折叠设备 → ratio fallback
                }
            }
    }
}

核心逻辑：

  • FoldingFeature.State.FLAT → 完全展开 → isFolded = false
  • State.HALF_OPENED → 半开 → isFolded = true
  • 无 FoldingFeature（普通手机/平板）→ 降级 ratio 判断

collectLatest 持续监听折叠变化，isFolded 通过 CompositionLocalProvider 注入 Compose 树，GalGameChatLayout 用 LocalIsFolded.current 读取后自适应 UI。
