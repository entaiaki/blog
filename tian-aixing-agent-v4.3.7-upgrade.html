折叠判断修复 + 桌宠触控极简化

2026-06-24

问题 1：fallbackDetectFolded 毁了全部设备

v4.3.5~v4.3.6 的 fallback 逻辑用了 ratio > 1.0f：

设备ratio结果
手机竖屏~0.5正常
手机横屏>1误判"折叠"
平板>1误判"折叠"
折叠外屏>1正确（巧合）

所有横屏设备都被当成了折叠外屏，UI 全坏。

修复：三状态 ScreenMode + 合理阈值

enum class ScreenMode {
    INNER_EXPANDED,   // 内屏展开 → 完整 UI
    OUTER_COMPACT,    // 外屏/折叠 → 舞台窗口
    UNKNOWN
}

private fun detectScreenMode(): ScreenMode {
    val ratio = dm.widthPixels / dm.heightPixels
    return when {
        ratio < 0.8f  -> INNER_EXPANDED   // 竖屏
        ratio <= 1.4f -> INNER_EXPANDED   // 近正方形
        ratio > 1.4f  -> OUTER_COMPACT    // 明显横向
        else          -> UNKNOWN
    }
}

关键：只有 ratio > 1.4 才算外屏。平板/手机横屏通常在 1.3~1.8，但折叠外屏更窄更长（~2.0+）；1.4 是合理的折中点。

WindowInfoTracker 路径优先：有 FoldingFeature 时直接用 HAL 层状态（FLAT vs HALF_OPENED），ratio fallback 仅用于非折叠设备。

外屏 UI：FoldAwareContainer + 舞台窗口

@Composable
fun FoldAwareContainer(content: @Composable () -> Unit) {
    val mode = LocalScreenMode.current
    when (mode) {
        OUTER_COMPACT -> {
            Box(Modifier.fillMaxSize(), contentAlignment = Center) {
                Box(
                    Modifier.fillMaxHeight().width(320.dp),
                    contentAlignment = Center
                ) { content() }
            }
        }
        else -> Box(Modifier.fillMaxSize()) { content() }
    }
}

外屏角色渲染改为：fillMaxHeight().width(280.dp) + ContentScale.Crop + Alignment.Center。不再是全屏铺满——变成了"舞台窗口"。

问题 2：桌宠触控过度设计

v4.3.6 的 OverlayTouchController 有 7 个回调 + 3 种交互（拖拽/点击/长按）+ 二次确认——但用户只需要两件事：

  1. 点 X → 关闭桌宠
  2. 点角色 → 打开 App

修复：极简 setOnTouchListener

composeView.setOnTouchListener { _, event ->
    when (event.action) {
        MotionEvent.ACTION_UP -> {
            if (isInCloseZone(event.x, event.y)) {
                dismissPetWindowOnly()    // X → 关闭
            } else {
                openMainApp()             // 角色 → 回 App
            }
        }
    }
    true
}

8 行代码替代了 OverlayTouchController（113 行）、FloatingTouchController（118 行）、GestureDetector、二次确认 toast 等全部组件。
