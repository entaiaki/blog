一、悬浮聊天框：从「只读」到「可输入」

v4.3 的悬浮聊天框只能看欢迎语——没有输入框，不接 LLM。v4.3.2 彻底重写：

1.1 ChatOverlayController — 焦点 + 键盘

Flag旧新含义
FLAG_NOT_FOCUSABLE✅❌ 移除阻止窗口获得输入焦点 → TextField 点不了
FLAG_LAYOUT_NO_LIMITS❌✅允许窗口延伸到屏幕边缘外、键盘区域外
SOFT_INPUT_ADJUST_RESIZE❌✅键盘弹出时压缩布局，不挡输入框
 
val params = WindowManager.LayoutParams(
    MATCH_PARENT, MATCH_PARENT,
    TYPE_APPLICATION_OVERLAY,
    FLAG_LAYOUT_IN_SCREEN or FLAG_LAYOUT_NO_LIMITS,  // ← 关键
    PixelFormat.TRANSLUCENT
).apply {
    gravity = Gravity.CENTER
    softInputMode = SOFT_INPUT_ADJUST_RESIZE             // ← 关键
}

1.2 ChatOverlay — 完整聊天 UI

从 v4.3 的「欢迎语 + 静态卡片」变为：

  • LazyColumn 消息历史（user / assistant 气泡）
  • OutlinedTextField 多行输入框
  • 发送按钮 → PlannerAgent.create(context, history).chatWithEmotion(text)
  • 情绪同步 → CharacterStateHolder.emotion = result.emotion
  • Loading 状态 + 错误处理

这样桌宠点击出来的聊天框就是 真正的 LLM 对话，不用切回 App。

二、桌宠右上角 X 按钮

之前关桌宠只能：长按 → AlertDialog → 点「关闭桌宠」。三步操作太绕。

v4.3.2 加 onClose 参数，右上角放 26dp IconButton（Close 图标）：

Box(modifier = Modifier.size(90.dp, 160.dp)) {
    // 角色图 + 动画 + 触摸
    Box(modifier = Modifier.fillMaxSize().pointerInput(...) { ... }) {
        CharacterBackground(...)
    }
    // 右上角 X
    IconButton(
        onClick = onClose,
        modifier = Modifier.align(Alignment.TopEnd).size(26.dp)
    ) {
        Icon(Icons.Default.Close, "关闭桌宠", tint = Color.White)
    }
}

交互行为明确分层：

操作效果
点桌宠身体弹出悬浮聊天框（InteractionBridge）
点聊天框右上角 X只关聊天框，桌宠还在
点桌宠右上角 X关桌宠 + stopSelf
长按桌宠AlertDialog（关闭桌宠 / 打开 App）
