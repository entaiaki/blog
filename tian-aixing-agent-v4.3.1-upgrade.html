为支持混合 png/jpg，CharacterPack 新增 assetExtension 字段：

data class CharacterPack(
    ...
    val assetExtension: String = "png"  // ← 新增，illust1/2 设为 "jpg"

fun assetPath(emotion: CharacterEmotion): String {
    val target = if (emotion in supportedEmotions) emotion else CharacterEmotion.IDLE
    return "characters/$folder/${target.name.lowercase()}.$assetExtension"
}

二、桌宠拖拽三连修复

Bug 1：Y 轴方向反了

原代码 params.y - dy — 手指向下拖，窗口反而向上飞。屏幕坐标系 Y↓，手势 delta 也是 Y↓，应该是加法：

// ❌ 旧
params.y = params.y - dy.toInt()

// ✅ 修
params.y = params.y + dy.toInt()

Bug 2：惯性动画子线程调用 WindowManager

FloatingTouchController 的惯性衰减跑在 Thread { ... } 里，直接回调 onMove → windowManager.updateViewLayout() 在子线程必崩。

val mainHandler = Handler(Looper.getMainLooper())
val touchController = FloatingTouchController(
    onMove = { dx, dy ->
        mainHandler.post {
            params.x = params.x + dx.toInt()
            params.y = params.y + dy.toInt()
            windowManager.updateViewLayout(composeView, params)
        }
    },
    ...
)

Bug 3：ComposeView 不响应触控

显式设 isClickable = true / isFocusable = true：

val composeView = ComposeView(this).apply {
    isClickable = true
    isFocusable = true
    ...
}

三、InteractionBridge 交互桥

把「桌宠点击」和「聊天框弹出」解耦：

object InteractionBridge {
    var onChatRequest: (() -> Unit)? = null
    fun requestChat() { onChatRequest?.invoke() }
}

// MainActivity.onCreate
InteractionBridge.onChatRequest = {
    ChatOverlayController.show(applicationContext)
}

// DesktopPetView 点击 → 不再直调 ChatOverlayController
// → InteractionBridge.requestChat() → Activity 注册的回调

好处：Service 不持有 Activity 引用，避免生命周期泄漏。

四、关于“聊天没反应”

ChatViewModel 从 v3.x 就用 流式 SSE 架构：LLMClient.chatStream() 逐字吐出 + EmotionParser 末尾解析 + Kokoro TTS 实时合成。并没有降级为单次 chatWithEmotion() 调用。

如果设备上「发消息没回复」，排查方向：

  1. 设置页 API Key 是否已填写
  2. API 地址是否可达（网络 / 代理）
  3. logcat 中搜索 ChatVM 看异常
